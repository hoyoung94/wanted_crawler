# 원티드(Wanted) 채용공고 크롤러 기획서 초안

---

## Context

사람인 크롤러(recruit_crawler)의 구조를 기반으로, 원티드(wanted.co.kr) 채용공고를 자동 수집하는 Python 크롤러를 제작한다.  
원티드는 공식 REST API(`/api/v4/jobs`)를 제공하므로 HTML 파싱 없이 JSON 응답만으로 깔끔하게 데이터를 수집할 수 있다.  
키워드·직무 필터와 경력 필터를 지원하고, 결과를 CSV로 저장하며, GitHub Actions로 매일 자동 실행한다.

---

## 프로젝트 구조

```
wanted_crawler/
├── .github/

├── requirements.txt
├── output/
│   └── wanted_jobs_YYYYMMDD.csv
└── README.md
```

## 기술 스택

| 항목 | 선택 | 이유 |
|------|------|------|
| 언어 | Python 3.11+ | 참고 프로젝트와 동일 |
| HTTP | `requests` | 원티드 REST API 호출 |
| 데이터 처리 | `pandas` | CSV 저장, 중복 제거 |
| 스케줄링 | GitHub Actions | 매일 오전 9시(KST) 자동 실행 |

> BeautifulSoup 불필요 — 원티드는 JSON API 응답을 제공함

## 핵심 기능

### 1. API 기반 공고 수집

- **엔드포인트**: `GET https://www.wanted.co.kr/api/v4/jobs`
- **페이지네이션**: offset 방식 (`limit=20`, `offset`을 20씩 증가)
- **최대 수집 페이지**: 설정 파일에서 제어 (기본 10페이지 = 200건)

#### 주요 쿼리 파라미터

| 파라미터 | 설명 | 예시 |
|----------|------|------|
| `job_sort` | 정렬 방식 | `job.latest_order` (최신순) |
| `years` | 경력 (년수, -1=전체) | `0`(신입), `1~3`(1-3년) |
| `locations` | 지역 | `seoul`, `gyeonggi`, `all` |
| `tag_type_ids` | 직무 카테고리 ID | `518`(백엔드), `655`(프론트엔드) |
| `query` | 키워드 검색 | `Python`, `React` |
| `limit` | 한 번에 가져올 수 | `20` |
| `offset` | 시작 위치 | `0`, `20`, `40`... |

### 2. 공고 상세 정보 수집 (선택적)

- **엔드포인트**: `GET https://www.wanted.co.kr/api/v4/jobs/{job_id}`
- 목록 API에서 얻지 못하는 필드 보완:
  - `main_tasks` (주요 업무)
  - `requirements` (자격요건)
  - `preferred_points` (우대사항)
  - `benefits` (복리후생)

> 요청 수가 늘어나므로 `time.sleep(0.5~1초)` 필수 적용

### 3. 필터 설정 (`config.py`)

```python
# config.py 예시
KEYWORDS = ["Python", "Django", "FastAPI"]   # 키워드 검색어 (OR 조건)
JOB_CATEGORY_IDS = [518, 872]               # 직무 카테고리 ID (백엔드 등)
EXPERIENCE_YEARS = 0                         # -1=전체, 0=신입, 1~N=경력
LOCATIONS = "all"                            # 지역 (all / seoul / gyeonggi 등)
MAX_PAGES = 10                               # 최대 수집 페이지 수
FETCH_DETAIL = False                         # 상세 정보 추가 수집 여부
```

### 4. 수집 데이터 필드 (CSV 컬럼)

| 컬럼명 | 설명 | 출처 |
|--------|------|------|
| `job_id` | 공고 ID | 목록 API |
| `title` | 공고 제목(직무명) | 목록 API |
| `company` | 회사명 | 목록 API |
| `industry` | 산업군 | 목록 API |
| `location` | 근무지 (시/구) | 목록 API |
| `experience_from` | 최소 경력 (년) | 목록 API |
| `experience_to` | 최대 경력 (년) | 목록 API |
| `category` | 직무 카테고리 | 목록 API |
| `skills` | 요구 기술 태그 | 목록 API |
| `due_date` | 마감일 | 상세 API |
| `main_tasks` | 주요 업무 | 상세 API (선택) |
| `requirements` | 자격요건 | 상세 API (선택) |
| `preferred` | 우대사항 | 상세 API (선택) |
| `benefits` | 복리후생 | 상세 API (선택) |
| `url` | 공고 링크 | 조합 생성 |
| `crawled_at` | 수집 일시 | 크롤러 생성 |

### 5. 중복 제거

- `job_id` 기준으로 중복 제거 (`pandas.drop_duplicates`)
- 이전 실행 CSV가 존재하면 불러와서 신규 공고만 추가 (append 모드)

### 6. GitHub Actions 자동화

```yaml
# .github/workflows/crawler.yml
on:
  schedule:
    - cron: '0 0 * * *'   # UTC 0시 = KST 오전 9시, 매일 실행
  workflow_dispatch:        # 수동 실행 버튼

jobs:
  crawl:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
      - run: pip install -r requirements.txt
      - run: python wanted_crawler.py
      - uses: actions/upload-artifact@v4   # CSV를 Actions Artifact로 저장
        with:
          name: wanted_jobs
          path: output/
```

## 크롤러 핵심 로직 흐름

```
main()
 ├── config 로드 (키워드, 필터 설정)
 ├── fetch_job_list()
 │    ├── offset=0부터 MAX_PAGES까지 반복
 │    ├── GET /api/v4/jobs?query=...&years=...&offset=N
 │    ├── 응답 data[] 파싱
 │    └── next 링크 없으면 종료
 ├── (FETCH_DETAIL=True) fetch_job_detail()
 │    └── 각 job_id로 GET /api/v4/jobs/{id}
 ├── deduplicate() - job_id 기준 중복 제거
 └── save_csv() - output/wanted_jobs_YYYYMMDD.csv 저장
```

## 검증 방법

1. **로컬 테스트**: `python wanted_crawler.py` 실행 후 `output/` 디렉토리에 CSV 생성 확인
2. **데이터 검증**: CSV를 열어 job_id, 회사명, 직무, URL 등 필드 정상 여부 확인
3. **중복 제거 검증**: 동일 공고 ID가 두 번 이상 나타나지 않는지 확인
4. **필터 검증**: 키워드·경력 필터 적용 시 해당 조건 공고만 수집되는지 확인
5. **GitHub Actions 검증**: `workflow_dispatch`로 수동 실행 후 Artifact에서 CSV 다운로드 확인
6. **API 제한 확인**: rate limit 발생 시 sleep 값 조정 (기본 0.5초)
